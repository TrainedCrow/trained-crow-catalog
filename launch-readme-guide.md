# Launch README Guide

A launch README should help a stranger understand the product quickly without sounding like a giant sales page.

## Core Sections

- product name
- buyer promise
- who it is for
- what is included
- what happens after payment or signup
- safe-use disclaimer
- support route
- clear CTA

## Keep It Specific

Good launch copy says what the buyer gets and what problem it helps with. Avoid vague claims like "grow faster" unless the page explains how.

## Mark Product Status Clearly

If a product is not live yet, say coming soon or staged. Do not add a buy CTA until delivery is connected.
